## 三、D1 观众接受度算法

### 3.1 核心思路

用贝叶斯平滑处理小样本偏差，再通过搜索池百分位归一化。

### 3.2 子指标

```
────────────────────────────────────────────
子指标 1：收藏率 (Collect Rate)
────────────────────────────────────────────

原始值： CR_raw = count_collect / count_view

问题：播放量 < 1000 时极不稳定（1次收藏就能从0%跳到0.1%）

贝叶斯平滑：
  CR_smooth = (count_collect + α_c) / (count_view + β_c)

  其中 α_c = 品类平均收藏率 × 品类平均播放量
       β_c = 品类平均播放量

  品类参数（从搜索池计算，以手机数码类为例）：
    品类平均收藏率 c_avg = 0.008 (0.8%)
    品类平均播放量 v_avg = 40000
    α_c = 0.008 × 40000 = 320
    β_c = 40000

  实际效果：
    播放1000，收藏50 → CR_raw=5%，CR_smooth=(50+320)/(1000+40000)=0.9%
    → 避免了小样本虚高

    播放10万，收藏2000 → CR_raw=2%，CR_smooth=(2000+320)/(100000+40000)=1.66%
    → 大样本时逼近真实值

  最终得分：在搜索池中计算 CR_smooth 的百分位 → 映射到 0-10

────────────────────────────────────────────
子指标 2：互动率 (Engagement Rate)
────────────────────────────────────────────

权重设计（互动深度递增）：
  点赞 = 1 分（随手行为）
  评论 = 3 分（需思考打字）
  分享 = 5 分（主动背书）
  收藏 = 8 分（长期参考价值）

  ER_raw = (like×1 + comment×3 + share×5 + collect×8) / view

  贝叶斯平滑同收藏率逻辑：
  ER_smooth = (raw_score + α_e) / (count_view + β_e)

  百分位 → 0-10

────────────────────────────────────────────
子指标 3：传播力 (Virality)
────────────────────────────────────────────

  VIR = count_share / count_view

  分享是比点赞更强的信号——分享意味着"我愿意为这个内容背书"

  贝叶斯平滑同上
  百分位 → 0-10

────────────────────────────────────────────
```

### 3.3 综合公式

```
D1 = 收藏率 × 0.35 + 互动率 × 0.30 + 传播力 × 0.35
```

### 3.4 算法伪代码

```python
def score_D1(video, pool):
    """观众接受度评分"""
    
    # 从搜索池计算品类参数
    cat_avg_view = pool.median('count_view')
    cat_avg_collect_rate = pool.sum('count_collect') / pool.sum('count_view')
    cat_avg_engage_rate = pool.weighted_avg('engage_raw')
    cat_avg_share_rate = pool.sum('count_share') / pool.sum('count_view')
    
    # 贝叶斯平滑参数
    alpha_c = cat_avg_collect_rate * cat_avg_view  # 收藏先验
    beta = cat_avg_view
    
    # 子指标1：收藏率
    cr_smooth = (video.count_collect + alpha_c) / (video.count_view + beta)
    cr_score = percentile(cr_smooth, pool.cr_smooth_list) * 10
    
    # 子指标2：互动率（加权）
    raw_engage = (video.count_like * 1 + video.count_comment * 3 + 
                  video.count_share * 5 + video.count_collect * 8) / video.count_view
    alpha_e = cat_avg_engage_rate * cat_avg_view
    er_smooth = (raw_engage * video.count_view + alpha_e) / (video.count_view + beta)
    er_score = percentile(er_smooth, pool.er_smooth_list) * 10
    
    # 子指标3：传播力
    vir_raw = video.count_share / video.count_view
    alpha_s = cat_avg_share_rate * cat_avg_view
    vir_smooth = (video.count_share + alpha_s) / (video.count_view + beta)
    vir_score = percentile(vir_smooth, pool.vir_smooth_list) * 10
    
    # 综合
    D1 = cr_score * 0.35 + er_score * 0.30 + vir_score * 0.35
    
    return round(D1, 1)
```

## 四、D2 商业参考价值算法

### 4.1 核心思路

商业参考价值衡量"这部片子适不适合作为商业项目的参考案例"。信号来源：收藏率（"留着当参考"）、榜单表现、平台质量分、团队信誉。

### 4.2 子指标

```
────────────────────────────────────────────
子指标 1：收藏率分 (已由D1计算，权重不同)
────────────────────────────────────────────

  直接复用 D1 的 cr_score
  在 D2 中权重更高（收藏 = 参考意图）

────────────────────────────────────────────
子指标 2：榜单加成 (Rank Bonus)
────────────────────────────────────────────

  从 ranks 数组提取：

  rank_bonus = 0
  for rank in video.ranks:
      if rank.code == "monthlyRanking":
          # 月榜加成最高
          bonus = max(0, 3 - rank.index) * 1.5  # 月榜#1→+4.5, #3→+0
      elif rank.code in ["staffPicks", "ad", "digital"]:
          # 专业/品类周榜
          bonus = max(0, 10 - rank.index) * 0.5  # 周榜#1→+5, #10→+0
      elif rank.code == "creative_recommend":
          bonus = 2.0  # 创意推荐固定加成
      rank_bonus += bonus
  
  rank_bonus = min(rank_bonus, 5)  # 上限5分

────────────────────────────────────────────
子指标 3：平台质量分
────────────────────────────────────────────

  quality_score = video.quality / 5 * 10  # quality范围1-5，线性映射到0-10

────────────────────────────────────────────
子指标 4：badge 加成
────────────────────────────────────────────

  badge_map = {
      "recommend": 1.5,
      "monthly_rank": 2.0,
      "weekly_rank": 1.0,
      "staffPicks": 2.5,
  }
  badge_bonus = sum(badge_map.get(b, 0) for b in video.badges)
  badge_bonus = min(badge_bonus, 3)  # 上限3分

────────────────────────────────────────────
子指标 5：团队信誉分
────────────────────────────────────────────

  team_score 来自 D3（团队专业度），作为商业价值的参考因子

────────────────────────────────────────────
```

### 4.3 综合公式

```
D2 = 收藏率分 × 0.30
   + 榜单加成   × 0.25
   + 平台质量   × 0.20
   + badge加成  × 0.10
   + 团队信誉   × 0.15
```

### 4.4 算法伪代码

```python
def score_D2(video, pool, D3_score):
    """商业参考价值评分"""
    
    # 复用D1的收藏率百分位分
    cr_score = D1_collect_rate_score(video, pool)
    
    # 榜单加成
    rank_bonus = 0
    rank_weights = {
        "monthlyRanking": (1.5, 3),   # (系数, 衰减阈值)
        "staffPicks": (1.0, 5),
        "ad": (0.5, 10),
        "digital": (0.5, 10),
        "creative_recommend": (2.0, 1),
    }
    for rank in video.ranks:
        w = rank_weights.get(rank.code, (0.3, 10))
        rank_bonus += max(0, w[1] - rank.index) * w[0]
    rank_bonus = min(rank_bonus, 5)
    
    # 平台质量分
    quality_score = (video.quality or 3) / 5 * 10
    
    # badge加成
    badge_values = {
        "recommend": 1.5,
        "monthly_rank": 2.0,
        "weekly_rank": 1.0,
        "staffPicks": 2.5,
    }
    badge_bonus = 0
    for badge_name in video.get_all_badges():
        badge_bonus += badge_values.get(badge_name, 0)
    badge_bonus = min(badge_bonus, 3)
    
    # 综合
    D2 = (cr_score * 0.30 + 
          rank_bonus * 0.25 * 2 +      # rank_bonus范围0-5, ×2映射到0-10
          quality_score * 0.20 +
          badge_bonus * 0.10 * 3.33 +  # badge_bonus范围0-3, ×3.33映射到0-10
          D3_score * 0.15)
    
    # D2上限为10（榜单和badge可能导致溢出）
    return round(min(D2, 10), 1)
```

## 五、D3 团队专业度算法

### 5.1 核心思路

从作者和团队成员的平台统计数据，评估团队的专业水准。这是一个"间接信号"——不能直接评价作品质量，但能提供参考。

### 5.2 子指标

```
────────────────────────────────────────────
子指标 1：导演影响力 (Director Authority)
────────────────────────────────────────────

  从 author.userinfo.count 提取：

  authority_raw = 0
    
  # 粉丝数（对数归一化）
  follower_score = min(log10(author.count_follower + 1) / log10(100000), 1) * 10
  # 1万粉→6.0,  10万粉→8.0,  100万粉→10.0
  
  authority_raw += follower_score * 0.3
  
  # 推荐率（被推荐次数/作品总数）
  if author.count_article > 0:
      recommend_rate = author.count_recommend / author.count_article
      # >50% → 10分, 20% → 6分, <5% → 2分
      recommend_score = min(recommend_rate / 0.5, 1) * 10
      authority_raw += recommend_score * 0.4
  
  # 人气值（对数归一化）
  pop_score = min(log10(author.count_popularity + 1) / log10(10000000), 1) * 10
  authority_raw += pop_score * 0.3

────────────────────────────────────────────
子指标 2：团队完整度 (Team Completeness)
────────────────────────────────────────────

  关键角色清单（广告片标准配置）：
    核心创意：导演、编剧
    核心制作：摄影指导/摄影师、美术指导
    核心后期：剪辑师、调色师
    加分项：  声音设计、航拍、特效、灯光、造型
  
  计分：
    team_completeness = 0
    core_roles = {"导演","编剧","摄影指导","摄影师","美术指导","剪辑师","调色师"}
    bonus_roles = {"声音设计","混音师","航拍","特效","灯光","造型指导"}
    
    # 从 team 列表和 author.role+occupation 提取角色集合
    filled_roles = extract_roles(video.team, video.author)
    
    # 核心角色每缺1个扣1.5分
    for role in core_roles:
        if any(role in r for r in filled_roles):
            team_completeness += 1.5
    team_completeness = min(team_completeness, 7.0)
    
    # 加分角色每个+0.5分（上限2.0）
    bonus = sum(0.5 for role in bonus_roles 
                if any(role in r for r in filled_roles))
    team_completeness += min(bonus, 2.0)
    
    # 团队规模加成
    if video.team_user_count >= 20: team_completeness += 1.0
    elif video.team_user_count >= 10: team_completeness += 0.5

────────────────────────────────────────────
子指标 3：行业认可度 (Industry Recognition)
────────────────────────────────────────────

  recognition = 0
  
  # 金雀奖
  jinque_count = sum(m.count_jin_que or 0 for m in team_members)
  recognition += min(jinque_count, 3) * 1.5  # 上限+4.5
  
  # 认证描述含关键词
  prestige_keywords = ["金狮奖","金雀奖","获奖","代表作","学院","国际"]
  for member in team_members:
      desc = member.verify_description or ""
      for kw in prestige_keywords:
          if kw in desc:
              recognition += 0.5
              break
  
  recognition = min(recognition, 5)
  # 映射到0-10: recognition×2
────────────────────────────────────────────
```

### 5.3 综合公式

```
D3 = 导演影响力 × 0.35 + 团队完整度 × 0.40 + 行业认可度 × 0.25
```

## 六、D4 内容新鲜度算法

### 6.1 核心思路

新发布的作品在"时效性"维度上有优势，老作品因数据积累更多在 D1/D2 上已有体现。D4 作为独立维度，帮助发现"新锐好片"。

### 6.2 公式

```
────────────────────────────────────────────

  days_since_publish = (now - publish_time) / 86400
  
  新鲜度 = max(0, 10 - days_since_publish / 30)
  
  发布7天内  → 新鲜度 9.0+
  发布30天内 → 新鲜度 9.0
  发布90天内 → 新鲜度 7.0
  发布180天  → 新鲜度 4.0
  发布365天  → 新鲜度 0

────────────────────────────────────────────
```

**注意**：D4 是"可选的辅助维度"，不参与默认排序。主要用于"最新好片"类搜索场景。

## 七、综合评分公式

### 7.1 评分向量

```
每部片子的评分向量 S = [D1, D2, D3, D4]
                          观众  商业  团队  新鲜
                         接受  参考  专业  度
```

### 7.2 场景化综合分

```
不同场景使用不同的权重向量 W：

  默认（通用浏览）：
    W = [0.30, 0.35, 0.25, 0.10]

  广告商选团队：
    W = [0.15, 0.50, 0.30, 0.05]

  创作者学习：
    W = [0.25, 0.20, 0.40, 0.15]

  最新好片：
    W = [0.25, 0.25, 0.15, 0.35]

  综合分 = S · W = D1×w1 + D2×w2 + D3×w3 + D4×w4
```

### 7.3 不设单一固定综合分

```
❌ 不做：综合评分 8.3/10（一个无法解释的黑盒数字）

✅ 做：  D1 观众接受度 8.6  超越92%同品类
        D2 商业参考   9.2  超越97%同品类
        D3 团队专业   9.5  顶级配置
        D4 内容新鲜   7.0  发布90天

        用户自行判断哪个维度对自己更重要。
```

## 十、算法参数调优建议

### 10.1 贝叶斯平滑参数

```
品类参数应从搜索池动态计算，而非写死：

  每24小时刷新一次品类基准：
    cat_avg_view       = pool.median(count_view)
    cat_avg_collect_rate = pool.sum(count_collect) / pool.sum(count_view)
    cat_avg_share_rate   = pool.sum(count_share) / pool.sum(count_view)
    
  过滤条件：
    排除播放量 < 100 的作品（噪音）
    排除发布超过 365 天的作品（时效性偏离）
    至少 20 部作品才能计算品类基准
```

### 10.2 权重调优

```
初始权重（基于经验设定），上线后通过用户行为调优：

  调优信号：
    1. 用户搜索后点击的作品 → 高D1权重的搜索排序是否更有效？
    2. 广告商发起联系的作品 → 高D2权重的筛选是否更精准？
    3. 用户收藏的参考作品 → 哪个子指标与收藏行为相关性最强？

  A/B测试方案：
    Group A: D1×0.35 + D2×0.35 + D3×0.15 + D4×0.15
    Group B: D1×0.25 + D2×0.25 + D3×0.35 + D4×0.15
    → 对比搜索点击率、收藏转化率
```

### 10.3 冷启动处理

```
新作品（播放量 < 500）：
  · D1标记为"低置信度"→ 不参与按D1排序
  · D2仍可用（榜单/badge/团队数据不受播放量影响）
  · 给予"新锐加成"（D4权重临时提升）

无团队数据的作品：
  · D3 标记为"不可用"→ 不参与按D3排序
  · 综合分自动重新分配权重（D3的权重按比例分配给D1/D2）
```

## 十一、离线计算与缓存策略

```
触发时机：
  · 新作品发布 → 立即计算（此时仅有D3/D4，D1/D2需等数据积累）
  · 播放量每增加 5000 → 重新计算D1/D2
  · 获得新榜单/badge → 重新计算D2
  · 每24小时 → 全量刷新品类基准参数

缓存：
  · 评分结果缓存 Redis（TTL=1小时）
  · 品类基准缓存 Redis（TTL=24小时）
  · 搜索页展示列表用预计算评分（不实时算）

计算优先级：
  P0: 首页推荐/搜索结果页的 Top 100 → 预计算缓存
  P1: 用户在作品详情页展开评分 → 实时计算
  P2: 全量作品定期评分刷新 → 离线队列
```
